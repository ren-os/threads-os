# posting_schedule.md — 投稿スケジュール

---

## 基本設定

| 項目 | 設定値 |
|------|--------|
| 投稿数 | 5本/日 |
| タイムゾーン | Asia/Tokyo（JST） |
| 自動実行 | 毎朝 7:00（Windowsタスクスケジューラー） |

---

## 1日の投稿時間帯

| 時刻 | ターゲットの状態 | 推奨コンテンツ |
|------|----------------|--------------|
| 07:00 | 通勤・起床直後 | 共感投稿（「わかる」と思わせる） |
| 10:00 | 仕事の隙間 | ノウハウ（保存されやすい） |
| 13:00 | 昼休み | ストーリー（読み応えのある体験談） |
| 18:00 | 退勤・帰宅中 | 共感 or 知らないと損するシリーズ |
| 20:00 | 夜のリラックスタイム | CTA付きストーリー（行動しやすい） |

---

## 週次バランス設計（35本/週）

| 柱 | 週の本数目安 |
|----|------------|
| 体験談・ストーリー | 17〜18本 |
| 転職ノウハウ | 10〜11本 |
| 共感投稿 | 7本 |
| CTA付き投稿（上記に含む） | 3〜5本 |

---

## コンテンツ補充サイクル

| 項目 | 頻度 |
|------|------|
| posts.csv 補充 | 4日に1回（20本÷5本/日） |
| posted_log.md 更新 | 補充のたびに |
| hooks.md 使用済み移動 | 補充のたびに |
| 投稿パフォーマンス確認 | 週1回 |
| ネタ出しセッション | 週1回（次の20本を準備） |

---

## 月次スケジュールテンプレート

```
1週目：フリーター・既卒向けコンテンツ中心
2週目：面接・履歴書・職務経歴書のノウハウ中心
3週目：ブラック企業・ホワイト企業の見分け方中心
4週目：まとめ・成功談・次月予告
```

---

## 運用上の注意

- PCはシャットダウンせず**スリープ**で終了すること
  （スリープ解除タイマーが有効なため）
- タスクスケジューラー名：`ThreadsAutoPost`
- 毎日の自動実行ログは `logs/scheduler.log` に記録される
- 予約済み投稿は `logs/posted_log.csv` で管理される

---

## 実行コマンド（手動で実行する場合）

```powershell
cd C:\Users\shota\threads-os
$env:PYTHONUTF8 = "1"
python main.py --dry-run    # テスト確認
python main.py --execute    # 本番実行
```
