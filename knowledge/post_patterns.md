# post_patterns.md — 勝ち投稿テンプレートDB

競合リサーチ・自己分析で判明した「バズる投稿の構成パターン」を蓄積するファイル。
`research/winning_posts.md` で発見した勝ち投稿から共通点を抽出し、ここに反映する。
`automation/generate_posts.md` の Step1 で必ず読み込む。

---

## 優先度ルール

```
★★★ 最優先：競合で3回以上同じ構成がバズっていたパターン
★★  高優先：競合で2回バズり、かつ自分の投稿でも好パフォーマンスだったパターン
★   通常：1回バズりを観測したパターン（仮説段階）

generate_posts.md での扱い：
・★★★ は今回の生成に必ず2本以上含める
・★★  は今回の生成に1本以上含める
・★   は参考程度。自分の経験談と組み合わせて使う
```

---

## テンプレート一覧

### Type-A｜問題提起型（★ → 観測で随時昇格）

**構成フロー：**
```
問題提起（ターゲットが今まさに持っている悩みを1文で）
↓
失敗例（「こうやってしまいがちなパターン」を具体的に）
↓
改善例（「こうすれば変わる」という具体的な方法）
↓
理由（なぜ改善例のほうが効果的なのか・根拠1〜2行）
↓
まとめ（「これを知っているかどうかで変わる」形式で締める）
```

**テンプレート文：**
```
[ターゲットの悩みを1行で]、ありませんか？

よくやってしまうパターン：
「[失敗例を具体的に]」

でも、こうするだけで変わります。
「[改善例を具体的に]」

理由は[シンプルな根拠]。

[まとめ：「これを知っているかどうかで、[具体的な結果]が変わります。」]
```

### Type-B｜Q&A型（★ → 観測で随時昇格）

**構成フロー：**
```
Q（ターゲットが実際に持っている疑問・不安をそのまま質問形式で）
↓
A（端的に・断言調で答える）
↓
Q（別の角度の疑問）
↓
A（根拠や体験談を1行添える）
↓
Q（最も重要な疑問・不安を最後に）
↓
A（最後だけ少し長めに・希望で終わらせる）
```

**テンプレート文：**
```
よく聞かれること、まとめて答えます。

Q：[疑問1]
A：[端的な回答]

Q：[疑問2]
A：[回答＋体験談1行]

Q：[疑問3]
A：[回答＋希望で締める]

他にも聞きたいことがあればコメントどうぞ。
```

### Type-C｜Before→After型（★ → 観測で随時昇格）

**構成フロー：**
```
Before（問題のある状態を「数字・具体的状況」で描く）
↓
変えたこと（1〜3つ。シンプルに列挙）
↓
After（改善した結果を「数字・具体的変化」で描く）
↓
理由・学び（「変えたのは〇〇だけだった」という形で締める）
```

**テンプレート文：**
```
[変化を予告するフック1行]

【Before】
[問題のある状態を具体的に・数字があれば使う]

↓ やったこと
・[変えたこと1]
・[変えたこと2（あれば）]

【After】
[改善した結果を具体的に・数字で]

変えたのは[シンプルな要素]だけでした。
```

### Type-D｜ランキング型（★ → 観測で随時昇格）

**構成フロー：**
```
テーマ宣言（「〇〇の○選、まとめます」または「〇〇のランキング」）
↓
①〜③（または①〜⑤）を番号付きで列挙
各項目に1行の理由・解説を添える
↓
最も重要な項目をハイライト（「特に①が重要」等）
↓
まとめ1行
```

**テンプレート文（3選）：**
```
[テーマ]のよくあるミス、3選。

①[項目名]
→ [理由・解説を1行で]

②[項目名]
→ [理由・解説を1行で]

③[項目名]
→ [理由・解説を1行で]

特に[①/②/③]が一番大きい。
[補足・まとめ1〜2行]
```

**テンプレート文（5選）：**
```
[テーマ]でやってよかったこと、5選。

①[項目名]（[効果1行]）
②[項目名]（[効果1行]）
③[項目名]（[効果1行]）
④[項目名]（[効果1行]）
⑤[項目名]（[効果1行]）

どれか1つでも参考になれば。
```

### Type-E｜コメント誘導型（★ → 観測で随時昇格）

**構成フロー：**
```
本文（ターゲットに「自分のことだ」と思わせる内容）
↓
問いかけ（「あなたはどうですか？」等のエンゲージメント誘発）
↓
「続きはコメントで」または「コメントで答えてください」
```

**テンプレート文：**
```
[共感を呼ぶ本文・体験談または悩み提示]

[問いかけ：「これ、あなたはどうですか？」]
コメントで教えてもらえると嬉しいです。
```

**または（続き誘導型）：**
```
[本文：前半だけ書いて情報を出し惜しみ]

続きはコメントで聞いてください。
1人1人に答えます。
```

## 優先度昇格ルール

```
以下の条件を満たしたら、パフォーマンスサマリーの「優先度」を更新する：

★ → ★★（高優先へ昇格）：
  競合で同じ構成を2回観測、かつ自分の同型投稿の平均パフォーマンス以上

★★ → ★★★（最優先へ昇格）：
  競合で同じ構成を3回以上観測（research/winning_posts.md のカウント）
  → 自動的に generate_posts.md での次回生成に「必ず2本以上」のフラグが立つ

降格（★★★ → ★★）：
  最優先パターンで自己投稿が3本連続で平均以下のパフォーマンスだった場合

更新タイミング：毎週月曜（automation/weekly.md タスク4）
更新作業：analysis/competitive_patterns.md → このファイル のパフォーマンスサマリー欄を書き換える
```

## knowledge/ への反映ルール

```
このファイルで ★★★ に昇格したパターンは：
→ threads/post_templates.md の「今月の優先型」に追加する
→ automation/generate_posts.md の生成条件に反映する

月次で使用頻度が高い型のパフォーマンスが下がった場合：
→ ランキングを見直し、次月の比率を調整する
```
