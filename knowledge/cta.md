# cta.md — CTA設計

---

## CTAの基本方針

**売り込まない。10投稿に1〜2本だけ。**

CTAは「信頼が積み上がった後の自然な流れ」として機能させる。
本文で価値を提供してから、最後に一言だけ案内する。

---

## LINE導線ルール

| ルール | 内容 |
|--------|------|
| 通常投稿のURL | **禁止**。本文にLINE URLを貼らない |
| CTA標準フレーズ | **「無料相談はプロフィールのリンクから」** で統一 |
| プロフィール | LINEリンクを設置済みとする |
| URL管理 | `knowledge/offer.md` の `LINE_URL` のみで管理。他ファイルに直書きしない |

---

## CTAの配置ルール

| ルール | 詳細 |
|--------|------|
| 頻度 | 10投稿に1〜2本 |
| 位置 | 投稿の最後（本文の締め） |
| 前提 | 本文が価値提供になっていること |
| 禁止 | 毎回CTA・本文より目立たせる・強引な誘導・URL直貼り |

## CTAフレーズ集

### 標準フレーズ（必ずこの形で統一）

```
無料相談はプロフィールのリンクから👇
```

### バリエーション（標準フレーズを崩さない範囲で）

```
無料相談はプロフィールのリンクから👇まず話すだけでもOKです。
無料相談はプロフィールのリンクから👇費用は一切かかりません。
無料相談はプロフィールのリンクから👇一人で悩まず気軽にどうぞ。
```

### ソフト導入系（標準フレーズと組み合わせる前置き）

```
何から始めればいいかわからない方は→
気になった方は→
同じ状況の方は→
（前置き）+ 「無料相談はプロフィールのリンクから👇」
```

## CTA投稿の構成テンプレート

### テンプレートA（体験談 + CTA）
```
[相談者ストーリー or 自分の体験談]

同じ状況の方は、無料相談はプロフィールのリンクから👇
```

### テンプレートB（FAQ + CTA）
```
Q. よく聞かれること。
→ 答え

Q. よく聞かれること。
→ 答え

無料相談はプロフィールのリンクから👇
```

### テンプレートC（ノウハウ + CTA）
```

もっと詳しく知りたい方は、

```

## やってはいけないCTA

| NG | 理由 |
|----|------|
| 毎投稿にCTAを入れる | 売り込み感が出て信頼を失う |
| 「今すぐ登録して」 | 煽りになる |
| 「登録しないと損」 | 不安商法になる |
| CTA前に本文がない | 信頼ゼロで誘導するのは効果なし |
| LINEのURLを本文に直貼り | スパム扱いされるリスク・URL変更時に全投稿が壊れる |
| 標準フレーズ以外の独自CTAを作る | 表現がバラけてブランドが定まらない |

---

## CTAの効果を高めるコンテキスト

以下の投稿タイプの後にCTAを入れると効果が高い。

1. **相談者の変化ストーリー**（「同じ状況の人へ」と自然につながる）
2. **FAQ投稿**（「もっと詳しく聞きたい」気持ちになっている）
3. **共感投稿の締め**（感情が動いている状態でCTAが入る）
