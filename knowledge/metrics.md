# metrics.md — KPI・計測設計

---

## 計測の目的

数値を見て「どの投稿が機能しているか」を把握し、
コンテンツ・導線・CTAを継続的に改善する。

---

## KPIの階層

```
北極星指標：LINE登録数（月）
↓
中間指標：プロフィール遷移率 / CTA投稿の転換率
↓
行動指標：閲覧数 / いいね率 / 保存率 / 返信率
```

## 週次レビューチェックリスト

- [ ] 今週の投稿の中で最もパフォーマンスが高かったのはどれか？
- [ ] 保存率が高い投稿のジャンル・型は何か？
- [ ] 返信が来た投稿はどれで、どんな内容の返信か？
- [ ] CTA投稿のLINE登録率はどうか？
- [ ] 次週に優先すべき投稿タイプは何か？

---

## 改善サイクル

```
投稿 → 計測（24〜48時間後） → 分析（週1回） → 改善（次回の投稿に反映）
```

**分析の優先順位**
1. 保存率が高い投稿 → ノウハウ系が刺さっているか確認
2. 返信率が高い投稿 → 共感ポイントを把握
3. プロフィール遷移が多い投稿 → CTA前の投稿タイプを特定
4. LINE登録が増えたタイミング → どの投稿が起点か
