# weekly.md — 毎週の実行内容

毎週月曜日に実施するルーティン。所要時間：60〜90分。
analysis/ のデータを元に改善を各フォルダへフィードバックする。

---

## 週次実行スケジュール（月曜日）

```
09:00 ── dashboard.md を更新（前週の数値入力）
         ↓
09:30 ── weekly_review.md を記入
         ↓
10:00 ── 勝ち投稿の抽出・viral_analysis.md を更新
         ↓
10:20 ── 競合チェック（competitor_watch.md）
         ↓
10:40 ── 改善アクションの実施（1〜2個）
         ↓
11:00 ── 翌週のposts.csv 残数を確認 → 不足なら投稿補充
         ↓
11:30 ── 週次タスク完了・reports.md に週次レポートを記録
```

---

## タスク1：dashboard.md 更新（09:00・15分）

```
実行手順：
1. analysis/dashboard.md を開く
2. 前週の各指標を入力する：
   - Threads：投稿数・インプレッション・いいね率・保存率・プロフィール遷移
   - LINE：友達追加数・ブロック数・フォーム誘導数
   - フォーム：表示数・完了数・相談予約数
   - ASP：紹介数・承認数・報酬額
   - ブログ：PV・主要キーワード順位
3. アラート状況欄（✅⚠️🔴）を更新する
4. alerts.md のルールに照らし合わせてアラートを確認する
```

---

## タスク2：週次レビュー（09:30・20分）

```
実行手順：
1. analysis/weekly_review.md のテンプレートをコピーして記入する
2. 今週よかった点・悪かった点・気づきを3つずつ書き出す
3. 翌週やることを3つ以内に絞る
4. アラートが出ている指標があれば最優先アクションを決める
```

---

## タスク3：勝ち投稿の抽出（10:00・10分）

```
実行手順：
1. threads/posted_log.md を開き、前週の投稿パフォーマンスを確認する
2. いいね・保存・プロフィール遷移が多かった投稿 TOP3 を特定する
3. analysis/viral_analysis.md の「週次バズチェック」に記入する
4. 勝ちパターンが明確なら：
   → threads/viral_patterns.md に追記する
   → threads/hooks.md に使えるフックがあれば追加する
5. 沈んだ投稿の共通点も確認し、analysis/viral_analysis.md の WORST3 に記入する
```

---

## タスク4：researchフォルダ週次まとめ（10:20・20分）

```
実行手順：
1. research/viral_archive.md を確認する
   → 今週のバズ投稿の共通パターンを抽出する
   → バズパターン集計テーブルを更新する

2. research/hooks_archive.md を確認する
   → 高パフォーマンスだったフックを knowledge/hooks.md に転記する
   → 「れんが使えるフック転用ストック」を補充する

3. research/comment_analysis.md を確認する
   → 今週よく出た悩みワード TOP3 を確認する
   → 3回以上来た質問を threads/ideas.md / line/faq.md に追記する

4. research/trend.md を確認する
   → 今週のトレンドサマリーを記入する
   → 来週の投稿テーマに反映すべきネタを threads/ideas.md に追加する

5. research/save_patterns.md を確認する
   → 今週の保存率集計テーブルを更新する
   → 翌週の投稿生成に反映する推奨型を記録する

6. research/competitor_tracker.md を確認する
   → 競合バズ投稿を確認・記録する
   → analysis/competitors.md にも同期する
```

---

## タスク5：週次改善アクション（10:40・20分）

weekly_review.md で決めたアクションを実施する。

```
改善の優先順位：
1. アラート（🔴）が発動している指標の改善
2. 週次レビューで「悪かった点」として挙げた項目
3. 勝ちパターンの他記事・他投稿への横展開

改善の実施ルール：
・1度に変える変数は1つ
・変更したらanalysis/improvement.md のログに記録する
```

### LINEの改善チェック（週次）

```
□ ブロック率が上昇していないか → line/broadcast.md の内容・頻度を確認
□ フォーム誘導クリックが減っていないか → step_messages.md のCTAを確認
□ 返信漏れがないか → LINEの未読を確認
```

### フォームの改善チェック（週次）

```
□ フォーム回答数が前週より減っていないか
□ 新着回答に24時間以内に返信できているか
□ Q5（自由記述）に新しいキーワードがあれば forms/responses.md に追記
```

### ASPの改善チェック（週次）

```
□ ASP管理画面で新着承認・否認を確認
□ 否認があれば asp/tracking.md の否認ログに記録
□ 登録後フォロー中のユーザーに進捗確認メッセージを送る
```

---

## タスク6：翌週の投稿準備（11:00・10分）

```
実行手順：
1. posts.csv の残数を確認する（python main.py --dry-run）
2. 残数 < 20本の場合 → generate_posts.md の投稿生成フローを実行する
3. 生成した投稿を posts.csv に追加する
4. threads/posted_log.md のセット番号を更新する
```

---

## タスク7：ブログ改善チェック（週次・月曜）

```
□ Googleサーチコンソールで主要キーワードの順位変動を確認する
□ 圏外に落ちた記事があれば blog_analysis.md にメモする
□ 今週公開した記事からのLINE登録クリック数を確認する
□ 翌週公開する記事の準備ができているか確認する
```

---

## 週次レポート記録

週次タスク完了後に reports.md の週次レポートテンプレートを記入する。
