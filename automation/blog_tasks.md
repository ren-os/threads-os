# blog_tasks.md — ブログ運用タスク定義

ブログの記事作成・SEO管理・改善に必要なタスクを定義する。

---

## 週次タスク（毎週月曜）

| タスク | 参照 | 所要時間 |
|-------|-----|---------|
| Googleサーチコンソールで順位変動確認 | analysis/blog_analysis.md | 5分 |
| 圏外記事をblog_analysis.mdにメモ | blog/update_rules.md | 3分 |
| 今週公開する記事の最終確認 | blog/checklist.md | 10分 |
| ブログ経由LINE登録クリック数を確認 | analysis/dashboard.md | 3分 |

---

## 月次タスク（毎月）

| タスク | 参照 | 所要時間 |
|-------|-----|---------|
| Googleアナリティクスで月次PV・滞在確認 | analysis/blog_analysis.md | 10分 |
| ブログKPIサマリーを作成 | blog/kpi.md | 10分 |
| リライト候補記事を特定する | blog/update_rules.md | 10分 |
| 翌月公開記事3〜4本のテーマ決定 | blog/ideas.md | 10分 |
| blog_analysis.md 月次テンプレートを記入 | analysis/blog_analysis.md | 20分 |
| 内部リンクの張り漏れを確認する | blog/internal_links.md | 10分 |

---

## 記事公開フロー（1記事あたり）

```
Step1：テーマ・キーワード決定
→ blog/ideas.md から「★★★優先」を選ぶ
→ 競合上位5記事の構成を確認する

Step2：記事執筆
→ blog/article_templates.md の対応テンプレートを使う
→ 文字数3,000字以上・H2は4〜7個

Step3：SEO設定
→ blog/seo.md のルールに沿ってタイトル・見出し・URL・メタを設定する
→ blog/internal_links.md に沿って内部リンクを3本以上追加する

Step4：CV導線設定
→ LINE誘導ブロックを本文中2回・末尾1回入れる
→ エージェントレビュー・比較記事はASPリンクも設置する

Step5：公開前チェック
→ blog/checklist.md の全項目をチェックする

Step6：公開後対応
→ Googleサーチコンソールでインデックスリクエスト
→ blog/kpi.md の記事リストに追加する
→ blog/ideas.md の「使用済みネタログ」に移動する
→ 関連する既存記事からこの記事へのリンクを追加する
```

---

## リライトフロー（1記事あたり）

```
Step1：対象記事の選定
→ blog/update_rules.md の「リライト優先度基準」で対象を決める

Step2：競合分析
→ 対象KWで検索して上位5記事の構成・文字数を確認する
→ 自記事との差分を洗い出す

Step3：リライト実施
→ H2の追加・文章の充実・内部リンク強化
→ 更新日を記事に反映する

Step4：リライト後の記録
→ blog/update_rules.md のリライト記録ログに追記する
→ Googleサーチコンソールで再インデックスをリクエストする
→ 1ヶ月後に順位変化を確認する
```

## ブログ初期設定タスク（一度限り）

```
□ ブログのプラットフォームを決定する（WordPress推奨）← TODO
□ ドメインを取得する ← TODO
□ Googleアナリティクス（GA4）を設定する
□ Googleサーチコンソールを設定する
□ プロフィールページを作成する（れんの経歴・実績）
□ カテゴリページ（HUBページ）を3カテゴリ分作成する
□ LINE登録リンクのbitly短縮URLを作成する
□ アフィリエイトリンクをブログに設定する
```
