# daily.md — 毎日の実行内容

REN OS の1日の実行ルーティンを定義する。
毎朝07:00の自動投稿実行を起点に、夜22:00の改善ログ記録まで流れる。

---

## 1日のタイムライン

```
07:00 ── 自動投稿実行（main.py --execute）
         ↓
08:30 ── SNSリサーチ（research/daily_research.md・15〜20分）
         ↓
09:00 ── LINE確認・返信
         ↓
10:00 ── 自動投稿実行（2本目）
         ↓
13:00 ── 自動投稿実行（3本目）+ フォーム確認
         ↓
18:00 ── 自動投稿実行（4本目）
         ↓
20:00 ── 自動投稿実行（5本目）+ コメント返信
         ↓
21:00 ── 当日コメント確認・research/comment_analysis.md に記録
         ↓
22:00 ── 改善ログ記録（気づきがあれば）
```

---

## タスク0：SNSリサーチ（08:30・15〜20分）

**目的：** 毎日の市場情報を収集して research/ に蓄積し、投稿生成に反映する。

```
実行手順：
1. research/daily_research.md を開いて手順に従う
2. Threads → X → TikTok → Instagram の順に確認する（各3〜5分）
3. 発見したものを以下に記録する：
   □ バズ投稿 → research/viral_archive.md
   □ 伸びたフック → research/hooks_archive.md
   □ コメント・悩み → research/comment_analysis.md
   □ トレンドワード → research/trend.md
4. 競合の動きがあれば research/competitor_tracker.md に記録する
```

**スキップ条件：** 記録すべき変化がない日はログを省略してOK。ただし必ず目視確認は行う。

---

## タスク1：Threads自動投稿（07:00・10:00・13:00・18:00・20:00）

**実行者：** Windows Task Scheduler（ThreadsAutoPost）が main.py を起動
**参照ファイル：** posts.csv

```
実行確認チェック：
□ posts.csv に当日分の投稿が入っているか（残数 ≥ 5本）
□ 残数が5本未満の場合 → generate_posts.md の手順で補充する
□ 投稿完了後、posted_log.md に記録されているか確認する
```

**残数アラート：** 残数が10本を下回ったら当日中に補充投稿を生成する

---

## タスク2：Threads投稿確認・コメント返信（20:00〜21:00）

**所要時間：** 10〜15分

```
実行手順：
1. Threadsアプリを開き、当日の5本の投稿を確認する
2. コメントが来ていたら threads/comments.md のテンプレートを使って返信する
3. コメント返信は30分以内を目標にする（アルゴリズム評価UP）
4. 特に反応が良かった投稿を viral_analysis.md の週次チェック欄にメモする
```

---

## タスク3：LINE確認・返信（09:00・必要に応じて随時）

**所要時間：** 10〜30分（問い合わせ量による）

```
実行手順：
1. LINEを開き、新着メッセージを確認する
2. 返信は line/reply_templates.md から適切なテンプレートを選ぶ
3. フォーム回答通知が来ていたら24時間以内に返信する（最優先）
4. 相談予約が入ったら asp/offer_flow.md のフェーズ1〜2を実行する
5. 未返信のまま24時間経過しそうな場合は alerts.md の L-6 を確認する
```

---

## タスク4：Googleフォーム確認（13:00頃）

**所要時間：** 5分

```
実行手順：
1. Googleフォームの管理画面を開き、新着回答を確認する
2. 新着回答がある場合 → line/reply_templates.md「フォーム回答後の最初の返信」を送る
3. forms/responses.md の蓄積テーブルに回答を追記する
4. 相談予約に至ったら asp/tracking.md の紹介ログに追記する
```

---

## タスク5：相談対応（予約がある日のみ）

**所要時間：** 30〜60分/件

```
実行手順：
1. 相談前日に line/reply_templates.md「相談当日のリマインド」を送る
2. 当日：asp/offer_flow.md のフェーズ2（ヒアリング〜案件紹介）を実行する
3. 相談後：asp/copywriting.md の「相談のまとめ」テンプレートでフォローアップを送る
4. asp/tracking.md の紹介ログを更新する
```

---

## タスク6：コメント確認・research記録（21:00頃）

**所要時間：** 5〜10分

```
実行手順：
1. 当日の5本の投稿コメントを全件確認する
2. 質問・悩みコメントを research/comment_analysis.md に追記する
3. 特定のフックに反応が多かった場合：
   → research/hooks_archive.md の評価欄を更新する
4. 平均より大きくバズった投稿があった場合：
   → research/viral_archive.md に記録する（翌日の週次前でもOK）
```

---

## タスク7：日次改善ログ（22:00頃）

**所要時間：** 5分（気づきがある日のみ）

```
記録内容：
・今日の投稿で特に反応が良かった/悪かったこと
・LINE返信で気づいたこと（よく来る質問・不安）
・相談で気づいたこと（ユーザーの実際の言葉）

記録先：
□ analysis/viral_analysis.md の週次チェックにメモ
□ threads/ideas.md に新しいネタとして追加（気づきがあれば）
□ asp/faq.md に新しい質問として追加（3回以上来た質問）
□ research/trend.md に今日の気づきトレンドを追記（あれば）
```

---

## 日次タスク優先順位

| 優先度 | タスク | 最低限やること |
|-------|-------|------------|
| 最高 | フォーム回答への返信 | 24時間以内に必ず返信 |
| 高 | LINE問い合わせへの返信 | 当日中に返信 |
| 高 | 投稿残数確認 | 残10本を切ったら補充 |
| 中 | コメント返信 | 当日中・なければスキップOK |
| 低 | 改善ログ | 気づきがあれば記録、なければスキップOK |

---

## 投稿残数が切れた場合の緊急フロー

```
posts.csv の残数 < 5本になった場合：

1. generate_posts.md の投稿生成フローを実行する
2. 20本の新規投稿を生成する
3. posts.csv に追加する
4. python main.py --dry-run で確認する
5. threads/posted_log.md にセット番号を更新する
```
