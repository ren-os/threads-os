# checklist.md — 運用漏れ防止チェックリスト

REN OS の日次・週次・月次で確認する全チェックリスト。

---

## 毎日チェック（5分）

```
□ Threads投稿が予定通り自動投稿されたか（07/10/13/18/20時）
□ LINEに未返信メッセージがないか
□ Googleフォームに未確認の回答がないか
□ Threadsのコメント・いいね・リプに未対応がないか
```

---

## 週次チェック（毎週月曜・90分）

### Threads

```
□ 先週の5投稿は全件投稿されたか
□ posts.csv に今後14日分（10本以上）のネタが残っているか
□ いいね100以上のバズ投稿があったか → analysis/viral_analysis.md に記録
□ フォロワー数の週次増減を dashboard.md に記入したか
```

### LINE

```
□ 先週の新規登録者数を dashboard.md に記入したか
□ ブロック率が急増していないか（+2%以上ならアラート）
□ フォーム誘導数を line_analysis.md に記録したか
□ ステップ配信は正常に動作しているか（TODO：設定後）
```

### フォーム

```
□ 先週の回答数を dashboard.md に記入したか
□ 回答完了率が55%以上か（下回るなら改善を検討）
□ フォーム → 相談予約 の転換があったか
```

### ASP

```
□ ASP管理画面で新着の承認・否認を確認したか
□ フォロー中ユーザーに進捗確認メッセージを送ったか
□ 紹介ログ（tracking.md）は最新になっているか
```

### ブログ

```
□ Googleサーチコンソールで順位の急落がないか
□ 今週公開予定の記事がある場合、checklist.md で確認したか
□ ブログ経由のLINE登録があったか
```

### 競合

```
□ 競合アカウント（C01〜C03）の先週投稿を確認したか
□ バズ投稿があれば competitors.md に記録したか
□ 転用アイデアを threads/ideas.md に追加したか
```

### 週次レポート

```
□ analysis/weekly_review.md を記入したか
□ reports.md の週次レポートを更新したか
□ 今週の改善アクション1〜2個を決めたか
```

## 月次チェック（毎月最終月曜・4時間）

### 月次KPI集計

```
□ Threads：フォロワー数・投稿数・平均エンゲージメント率を集計した
□ LINE：月次登録者数・ブロック数・フォーム誘導数を集計した
□ フォーム：月次回答数・完了率・相談予約率を集計した
□ ASP：紹介数・登録率・承認数・月間報酬を集計した
□ ブログ：月次PV・登録誘導数・ASPクリック数を集計した
□ analysis/dashboard.md の全数値を更新した
```

### 月次改善

```
□ analysis/monthly_review.md を記入した
□ 勝ちパターン・負けパターンを3つ以上抽出した
□ 各フォルダへのフィードバックを実施した（threads/line/forms/blog/asp）
□ knowledge/ の内容を最新情報に更新したか
□ improvement.md の「月次改善ログ」に追記した
```

### コンテンツ

```
□ 翌月の投稿を20本以上生成してposts.csvに追加した
□ 翌月のブログ記事テーマ（3〜4本）を決定した
□ LINE配信カレンダー（翌月分）を作成した
□ 使用済みネタが重複していないか確認した
```

### ASP・収益

```
□ ASP管理画面で月次承認チェックをした
□ 否認案件の原因分析をして selection_rules.md を更新した
□ 今月の月間報酬を reports.md に記録した
```

### TODO管理

```
□ automation/todo.md のP1タスクの進捗を確認した
□ 完了したタスクを「完了済み」に移動した
□ 新しいTODOが出た場合に追記した
□ P2タスクの中でP1に格上げすべきものがないか判断した
```

## 初期設定チェック（一度限り・起動時）

```
□ LINE公式アカウントを開設した
□ Googleフォームを作成・URLを設定した
□ フォームURLをLINEメッセージ全箇所に埋め込んだ
□ UZUZ のASP登録を完了した
□ キャリアスタートのASP登録を完了した
□ 全ASP案件のアフィリエイトリンクを取得・設定した
□ ブログプラットフォーム・ドメインを決定・設定した
□ Googleアナリティクス（GA4）を設定した
□ Googleサーチコンソールを設定した
□ 競合アカウント3〜5個を特定して competitor_watch.md に記入した
□ Googleカレンダーに週次・月次ルーティンを登録した
□ Windows Task Scheduler の実行ログを確認した（main.py が正常動作するか）
```

## アラート発動時の緊急チェック

```
🔴 フォロワーが前週比マイナスの場合
→ 直近の投稿の型・フック・内容を見直す（threads/templates.md）

🔴 LINE登録が週3件以下の場合
→ CTA投稿の本数・フォロワー数・プロフィールのLINEリンクを確認する

🔴 フォーム回答完了率が40%以下の場合
→ forms/questions.md の問数・質問文を見直す

🔴 ASP否認が連続3件の場合
→ asp/selection_rules.md のNG条件を見直す・ASPサポートに問い合わせる

🔴 ブログ記事が圏外（100位以下）の場合
→ blog/update_rules.md のリライトフローを実行する
```

---

## チェックリスト運用ルール

- 毎日チェックは朝・夕の2回確認する（各2〜3分）
- 週次チェックは必ず月曜日の09:00〜11:30で完結させる
- 月次チェックは最終月曜日に4時間ブロックして集中して行う
- チェックを飛ばした場合は翌日・翌週に必ず補完する
- 「確認した」だけでなく「数値を記録した」まで完了とする
