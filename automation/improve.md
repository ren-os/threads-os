# improve.md — 改善案フィードバック実行ルール

analysis/ で出た改善案を knowledge/ threads/ line/ forms/ blog/ asp/ へ
実際に反映するための実行手順を定義する。

---

## 改善反映の基本原則

1. **根拠を持って変える：** analysis/ の数値変化がトリガー。感覚での変更はしない
2. **1度に1変数：** 複数を同時に変えると改善効果の原因が特定できなくなる
3. **変えたら記録する：** analysis/improvement.md の改善ログに必ず記録する
4. **翌週・翌月に効果を検証する：** 変更後の数値変化を必ず追う

---

## 改善反映のトリガーと実行タイミング

| トリガー | タイミング | 参照元 |
|---------|---------|-------|
| アラート（🔴RED）発動 | 当日〜翌日 | analysis/alerts.md |
| 週次レビューで「悪かった点」特定 | 当週中 | analysis/weekly_review.md |
| 月次分析で勝ち/負けパターン確定 | 月次フィードバック実施日 | analysis/monthly_review.md |

## knowledge/ への反映ルール

| 更新条件 | 更新先ファイル | 反映内容 |
|---------|------------|---------|
| フォームQ3「その他」が30%超 | knowledge/faq.md | よく出るキーワードをQ&Aとして追加 |
| フォームQ1の分布が想定とズレる | knowledge/target.md | ペルソナの比重・属性の記述を更新 |
| 相談で同じ不安が3回以上来る | knowledge/objections.md | 反論設計を追加 |
| 案件の承認率データが更新された | knowledge/asp.md | エージェント優先順位を更新 |
| バズジャンルが変化した | knowledge/content_pillars.md | 投稿テーマの優先順位を更新 |

```
【実行手順】
1. analysis/ の対象ファイルを開いてデータを確認する
2. 上記テーブルの条件に当てはまるか判断する
3. 対象ファイルを更新する
4. analysis/improvement.md の改善ログに記録する
```

## 改善反映チェックリスト（実施のたびに確認）

□ 翌週・翌月の確認スケジュールを設定した
