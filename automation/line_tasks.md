# line_tasks.md — LINE運用タスク定義

LINE公式アカウントの運用に必要なタスクを日次・週次・月次で定義する。

---

## 日次タスク（毎日）

| 時間 | タスク | 参照 | 所要時間 |
|-----|-------|-----|---------|
| 09:00 | 新着メッセージ確認・返信 | line/reply_templates.md | 10〜30分 |
| 13:00 | Googleフォーム新着確認・24h以内返信 | line/reply_templates.md | 5〜10分 |
| 随時 | 相談予約者へのリマインド | line/reply_templates.md | 5分/件 |

```
日次確認チェック：
□ 未読メッセージがないか
□ フォーム回答通知が来ていないか
□ 相談予約日当日のリマインド送信が必要か
□ 7日以上無反応のユーザーへフォローが必要か
```

---

## 週次タスク（毎週月曜）

| タスク | 参照 | 所要時間 |
|-------|-----|---------|
| 友達追加数・ブロック数を記録 | analysis/dashboard.md | 5分 |
| ブロック率をアラートと照合 | analysis/alerts.md L-2 | 3分 |
| フォーム誘導クリック数を記録 | forms/kpi.md F5 | 3分 |
| ステップ配信の送信数を確認 | line/step_messages.md | 5分 |
| 無反応ユーザーへのフォローを実施 | line/reply_templates.md カテゴリ5 | 10分 |

---

## 月次タスク（毎月）

| タスク | 参照 | 所要時間 |
|-------|-----|---------|
| LINE月次KPIサマリーを作成 | line/kpi.md | 10分 |
| line_analysis.md の月次テンプレートを記入 | analysis/line_analysis.md | 20分 |
| broadcast.md の翌月配信カレンダーを作成 | line/broadcast.md | 10分 |
| step_messages.md の見直し（ブロック率が高ければ） | line/improvement.md | 15分 |
| faq.md に頻出質問を追加 | line/faq.md | 10分 |

---

## LINE設定タスク（初期設定・一度限り）

```
□ LINE公式アカウントを開設する
□ greeting.md の3通メッセージをステップ配信に設定する
□ step_messages.md の Day1〜7 をステップ配信に設定する
□ richmenu.md の6ボタンレイアウトを実装する
□ キーワード自動返信（よくある質問・体験談・自己紹介）を設定する
□ フォームリンクをbitly等で計測用短縮URLに変換して設定する
□ broadcast.md の定期配信（火曜20:00・土曜10:00）を設定する
```

---

## LINE改善チェック（週次・アラート確認時）

```
ブロック率 > 5% の場合：
□ 直近の broadcast.md 配信内容を確認する
□ step_messages.md のDay別内容を見直す
□ 配信頻度を週1回に下げることを検討する

フォーム誘導率 < 15% の場合：
□ step_messages.md Day3・5・7のCTAコピーを改善する
□ richmenu.md の①ボタンが正常に機能しているか確認する

相談予約率 < 30% の場合：
□ フォーム後の返信スピードを確認する（24時間以内か）
□ reply_templates.md「フォーム回答後の最初の返信」の文章を見直す
```

---

## ブロードキャスト配信管理（月次）

```
毎月末に翌月分の配信カレンダーを作成する：

【翌月配信カレンダー作成手順】
1. line/broadcast.md の月次カレンダーテンプレートをコピーする
2. 翌月の火曜・土曜の日付を入れる
3. A/B/C/Dのコンテンツタイプをローテーションで割り当てる
4. TYPE A（ノウハウ）のネタを threads/ideas.md や blog の記事から選ぶ
5. LINE公式アカウントに予約投稿を設定する
```
