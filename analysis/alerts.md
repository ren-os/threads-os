# alerts.md — KPIアラートルール

数値が悪化したときに「何を確認して・何をすべきか」を即座に判断できるルールブック。
dashboard.md の週次更新時に自動チェックする。

---

## アラートレベル定義

- 🔴 **RED（即対応）：** 目標の半分以下。当日〜翌日中に対処する
- ⚠️ **YELLOW（注意）：** 目標を下回っているが許容範囲内。今週中に改善案を検討する
- ✅ **GREEN（正常）：** 目標を達成または超えている

## 複合アラート（複数指標が同時に悪化した場合）

| 組み合わせ | 判断 | アクション |
|---------|-----|---------|
| T-3（いいね率）🔴 ＋ L-1（友達追加）🔴 | コンテンツ戦略全体の問題 | monthly_review.md の早期実施 → コンテンツ戦略を再構築 |
| L-2（ブロック率）🔴 ＋ L-5（相談率）🔴 | LINE上での信頼が低下している | line/step_messages.md・line/broadcast.md を全面見直し |
| A-4（承認率）🔴 ＋ A-5（3件連続否認）🔴 | 案件選定または紹介前確認の問題 | asp/selection_rules.md のNG条件を緊急強化 |
| B-4（圏外）🔴 ＋ B-5（CVR）🔴 | 記事品質とCTA両方に問題 | 対象記事をリライト＋LINE誘導を全面見直し |

## 週次アラートチェックフロー

毎週月曜日に dashboard.md を更新した後、以下を実施する。

```
1. dashboard.md の各指標を確認する
2. alerts.md のルール表と照合する
3. 🔴 RED があれば当日中に対応する
4. ⚠️ YELLOW は今週中に改善案を1つ選んで実施する
5. アラート発動ログに記録する
```
