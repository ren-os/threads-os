## パターン更新ルール

### 更新タイミング
- 週1回（毎週日曜）
- 20本分のデータが溜まるごと

### 更新手順

```
1. 直近20本の投稿データを集計する
2. 上位5本・下位5本を特定する
3. 上位5本の共通点を抽出する
4. 下位5本の問題点を特定する
5. 改善案を threads/improvement.md に追記する
6. 次回の優先パターンを更新する
```

### 優先パターンの変更基準

| 条件 | アクション |
|------|---------|
| ある型の保存率が他より2倍以上高い | その型の使用頻度を増やす |
| ある型の閲覧数が平均の50%以下 | その型の使用頻度を下げる |
| 特定のジャンルの反応が高い | そのジャンルのネタを増やす |
