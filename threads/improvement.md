# improvement.md — 改善サイクル管理

投稿パフォーマンスの分析結果を受けて、次回の生成・運用に反映する改善案を管理する。
分析元データは `threads/analyze_posts.md` を参照すること。
KPI進捗は `threads/kpi.md`、A/Bテスト結果は `threads/ab_test.md` を参照すること。

---

## 改善サイクルの全体像

```
投稿（posts.csv）
    ↓（24〜48時間後）
データ収集（analyze_posts.md / kpi.md）
    ↓（週1回・日曜）
分析・改善案抽出（analyze_posts.md → improvement.md）
A/Bテスト判定（ab_test.md → 勝者確定 → improvement.md に追記）
KPI進捗確認（kpi.md → 目標乖離があれば改善アクション追記）
    ↓（次回生成時）
改善反映（generate_posts.md が improvement.md / ab_test.md / kpi.md を読み込む）
    ↓
次の20本を生成
```

---

## 次回生成への反映事項（最新）

投稿を生成する前に必ずここを確認し、反映してから生成すること。

```
【最終更新日】2026-06-15（初期設定）

【次回反映事項】
1. T02（チェックリスト型）とT04（ランキング型）をそれぞれ2本以上入れる
   ← 直近20本で少なかったため
2. T08（あるある型）を1本試す
   ← 未使用のため効果検証が必要
3. フリーターと既卒に偏りすぎず、第二新卒と派遣・契約社員向けネタを増やす
   ← ターゲット全属性を網羅するため
```

## 常設の改善ルール

データが蓄積されていない現時点でも適用する基本改善方針。

### フック改善
- 冒頭20文字以内を維持する
- 数字を入れるとクリック率が上がりやすい
- 同じパターン（「知らないと損する〜」等）を3本以上連続させない

### 本文改善
- 1行が長くなりすぎたら2行に分割する
- 箇条書きは3〜5項目に絞る
- ストーリーは「転機」「結果」に具体的な数字を入れる

### CTA改善
- CTA投稿の前後に「信頼構築投稿（ストーリーorノウハウ）」を置く
- CTAフレーズを毎回変える（ローテーションは `threads/cta.md` 参照）
