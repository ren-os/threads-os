# posting_checklist.md — 投稿前チェックリスト

投稿をposts.csvに登録する前に全項目を確認すること。
1つでも×があれば修正してから登録する。

---

## チェックリスト（全投稿共通）

### A. 基本ルール

| # | チェック項目 | OK | NG |
|---|------------|----|----|
| A1 | 文字数が150〜350文字の範囲内か | ✓ | 修正 |
| A2 | 冒頭1行がフック（続きを読みたくなる）になっているか | ✓ | 修正 |
| A3 | 改行が適切に入っているか（詰まっていないか） | ✓ | 修正 |
| A4 | 専門用語が使われていないか | ✓ | 修正 |
| A5 | 読んで意味がわかるか（中学生レベルの言葉か） | ✓ | 修正 |

### B. NGコンテンツチェック

| # | チェック項目 | OK | NG |
|---|------------|----|----|
| B1 | 「絶対に」「必ず」等の成果保証表現がないか | ✓ | 削除 |
| B2 | 「今すぐ」「急いで」等の煽り表現がないか | ✓ | 削除 |
| B3 | 「楽して」「簡単に」等の誇大表現がないか | ✓ | 削除 |
| B4 | 具体的な会社名・サービス名が出ていないか | ✓ | 削除 |
| B5 | 特定の会社をブラック企業と断定していないか | ✓ | 修正 |
| B6 | 「転職すべき」「辞めるべき」等の強制表現がないか | ✓ | 修正 |
| B7 | 嘘・誇大表現がないか | ✓ | 削除 |

### C. 重複チェック

| # | チェック項目 | OK | NG |
|---|------------|----|----|
| C1 | `threads/posted_log.md` のフックと被っていないか | ✓ | 変更 |
| C2 | `threads/hooks.md` の使用済みリストと被っていないか | ✓ | 変更 |
| C3 | 直近10本で同じテーマを扱っていないか | ✓ | ネタ変更 |
| C4 | 直近2本で同じ型（T01〜T10）を使っていないか | ✓ | 型変更 |

### D. CTA確認（CTA付き投稿のみ）

| # | チェック項目 | OK | NG |
|---|------------|----|----|
| D1 | 本文でしっかり価値提供できているか | ✓ | 本文修正 |
| D2 | CTAは投稿の最後に自然な流れで入っているか | ✓ | 修正 |
| D3 | 今日・この週ですでに2本以上CTA投稿を入れていないか | ✓ | 削除 |
| D4 | ハードルを低く見せているか（「見るだけでもOK」等） | ✓ | 修正 |

### E. ターゲット確認

| # | チェック項目 | OK | NG |
|---|------------|----|----|
| E1 | ターゲット（18〜34歳・第二新卒・フリーター等）に刺さるか | ✓ | 修正 |
| E2 | ターゲットの悩みに直接触れているか | ✓ | 修正 |
| E3 | 「この人に相談してみたい」と思わせる内容か | ✓ | 修正 |

### F. 投稿比率バランス確認（20本全体）

| # | チェック項目 | OK | NG |
|---|------------|----|----|
| F1 | 体験談・ストーリーが10本（50%）になっているか | ✓ | 調整 |
| F2 | 転職ノウハウが6本（30%）になっているか | ✓ | 調整 |
| F3 | 共感投稿が4本（20%）になっているか | ✓ | 調整 |
| F4 | CTAが2本（10%相当）になっているか | ✓ | 調整 |
| F5 | 「知らないと損するシリーズ」が1本以上入っているか | ✓ | 追加 |

## 自動チェックスクリプト仕様（将来実装用）

# 将来実装する自動チェック項目

def check_post(content):
    checks = {
        "文字数": 150 <= len(content) <= 350,
        "NG表現なし": not any(ng in content for ng in NG_WORDS),
        "重複なし": check_duplicate(content),
        "会社名なし": not contains_company_name(content),
    }
    return checks
