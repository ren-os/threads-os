# post_templates.md — 投稿テンプレート集

投稿を生成する際に参照するテンプレート定義書。
型を使い分け、同じ型を連続して使わないこと。

## T01：ストーリー型

```
[フック：状況設定]（続きを読みたくなる1行）

[状況]（読者が自分と重ねられる2〜3行）

[転機・きっかけ]

[行動・変化]（具体的に）

[結果]（数字があれば入れる）

[学び or メッセージ]（1〜2行）
```

**ルール**
- 数字を入れる（「3ヶ月で」「30社落ちて」）
- 失敗→改善→成功の流れが最も刺さる
- 最後は「読者への問いかけ or メッセージ」で締める

**例文構造**
```
25歳フリーター、内定まで3ヶ月かかった話をします。

最初は「自分なんか無理」と思ってた。
バイト歴5年。正社員経験ゼロ。

変えたのは3つだけ。（箇条書き）

3ヶ月後、内定。

フリーター期間は「なかったこと」じゃなくて
「どう見せるか」の問題です。
```

## T02：チェックリスト型

```
[フック：リスト予告]

□ 項目1
□ 項目2
□ 項目3
□ 項目4（最大5個）

[まとめ or メッセージ]（1〜2行）
```

**ルール**
- 「当てはまる人いませんか？」の問いかけを入れる
- 3〜5項目（多すぎない）
- 最後に「〜な人はプロフィールから」でCTA化できる

---

## T03：NG vs OK 比較型

```
[フック：比較予告]

NG：「〇〇」
OK：「〇〇」

[なぜ違うのかを1〜2行で解説]

[まとめ]（「内容は同じでも印象は変わる」等）
```

**ルール**
- NG → OK の順番で書く（解決策で終わる）
- 理由を必ず添える（「なぜOKなのか」）
- 保存を促す実用性を意識する

---

## T04：ランキング型

```
[フック：ランキング予告]

①〇〇
②〇〇
③〇〇
（最大5位）

[補足 or 特に重要な項目の解説]

[まとめ or メッセージ]
```

**ルール**
- 「意外な内容」か「知らなかった」と思わせる内容を入れる
- 項目は短く（1行以内）
- 最後に「特に〇番が重要」と絞ると読まれやすい

---

## T05：FAQ型

```
[フック：「よく聞かれる」宣言]

Q. 〇〇
→ 〇〇（短く答える）

Q. 〇〇
→ 〇〇

Q. 〇〇
→ 〇〇

[まとめ or CTA]
```

**ルール**
- 2〜4問（多すぎない）
- 答えは1〜2行で簡潔に
- CTA前に置くと効果大

---

## T06：共感一言型

```
[共感フック]（1行。読者の心の声を代弁する）

[共感の深掘り]（「その気持ち、わかります」から展開）

[安心感 or 前向きメッセージ]（否定しない）

[余韻]（次のアクションを押しつけない）
```

**ルール**
- 読者の「言語化できなかった気持ち」を代弁する
- 解決策を急がない（共感で終わっていい）
- 「それ普通のことです」系の安心フレーズが刺さる

---

## T07：知らないと損するシリーズ

```
知らないと損する転職の話。

[テーマの一言提示]

[なぜ知らないと損するのかの背景]

NG例 or 問題のある状況：
→ 具体的に

OK例 or 解決策：
→ 具体的に

[まとめ1行]
```

**ルール**
- 必ず「知らないと損する転職の話。」で始める（シリーズ認知）
- NG→OK構造で実用的にする
- 保存率を最優先に設計する

---

## T08：あるある型

```
[フック：あるある予告]

あるある①
あるある②
あるある③

[共感メッセージ or 「これ、普通じゃないです」系の気づき]
```

**ルール**
- 「そうそう！」と思わせるリアルな描写
- ブラック企業・転職あるあるが刺さりやすい
- いいね・リポストを狙う型

---

## T09：ビフォーアフター型

```
[フック：変化の予告]

【Before】
〇〇な状態だった（具体的に）

↓ やったこと

【After】
〇〇になった（具体的に・数字あれば）

[学び or メッセージ]
```

**ルール**
- Before/After を視覚的に分ける
- 変化の具体性が命（「なんとなく良くなった」はNG）
- 職務経歴書・履歴書・面接対策ネタに使いやすい

---

## T10：CTA型

```
[本文]（T01〜T09の型でしっかり価値提供）

[締め：1〜2行で自然なCTA]
```

**ルール**
- 必ず本文で価値提供してからCTAを入れる
- CTA単独では立てない
- フレーズは `knowledge/cta.md` から選ぶ
- ハードルを最低にする（「見るだけ」「30秒だけ」）
