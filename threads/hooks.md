## フックの役割

冒頭1行でスクロールを止める。
「続きを読む」を押させた数 ＝ 閲覧数の差となる。

## フック改善ルール

### 改善トリガー
以下の条件を満たしたフックは「改善対象」とする。

- 閲覧数が直近20本の平均の50%以下
- いいね率が0.5%未満
- 保存率が0.1%未満

### 改善の手順

1. 低パフォーマンスフックのパターンを抽出する
2. 高パフォーマンスフックとの差分を分析する
3. 改善案を `threads/improvement.md` に追記する
4. 次回生成時に改善案を反映する

### 改善の方向性

| 問題パターン | 改善方向 |
|------------|--------|
| 冒頭が長すぎる | 20文字以内に圧縮する |
| 「〜について書きます」系 | 具体的な数字・状況に変える |
| 「〜ですよね」の多用 | 別の共感フレーズに変える |
| 同じ冒頭パターンの連続 | フックの型を変える |

## フック生成プロンプト

新しいフックが必要な場合、以下の公式で生成する。

```
① 数字系
「[数字][期間/量]で[結果]した話をします。」
「[数字]社落ちた後に変えたこと。」

② 共感系
「[読者の心の声]、[共感の一言]。」
「「[ターゲットの言葉]」と思ってる人へ。」

③ 知識系
「知らないと損する[テーマ]の話。」
「[テーマ]で[失敗/損]する人の特徴。」

④ 参加型
「[条件]の人、見てください。」
「これ、当てはまる人いませんか？」

⑤ 逆説系
「[常識]は間違いです。」
「[経験]は[ハンデ]じゃないです。」
```

## 未使用フックストック

※ここに常に20本以上をストックしておくこと。

### ストーリー系
- 面接で3回連続落ちた後、変えたことがあります。
- 「フリーターは無理」と思い込んでいた24歳の話。
- 退職理由を正直に言いすぎて3社落ちた話。
- 相談してから2ヶ月で内定が出た話、聞きますか。
- 派遣社員4年目、正社員になれた話をします。
- 「もう諦めよう」と思った翌月に内定が出た話。
- 契約更新を断られる1週間前に動き出した話。
- 転職先を1週間で決めて後悔した人の話。
- 履歴書を10回書き直してやっと通った話。
- 面接官に「なぜ短期離職したんですか」と聞かれた時の話。

- 知らないと損する転職の話。（内定承諾前の確認事項）
- 知らないと損する転職の話。（転職回数が多い人の対策）
- 知らないと損する転職の話。（給与交渉のタイミング）
- 知らないと損する転職の話。（試用期間中の退職）

### 共感系
- 転職したいのに、動けない。
- 毎朝、会社に行きたくないと思っている人へ。
- 「フリーターって言いたくない」と思ってる人へ。
- 面接が終わるたびに、また落ちた気がする。
- 「どうせ自分なんか」と思ってる人に伝えたいことがある。

### チェックリスト・ランキング系
- 転職前に確認すべきこと、チェックしてみてください。
- 第二新卒の転職でやりがちな3つのミス。
- 面接でやってはいけないこと、正直に書きます。
- 転職成功した人と失敗した人の違い、まとめます。
- 面接でよく聞かれる質問、TOP5。

### FAQ系
- フリーター転職でよくある質問に答えます。
- 「第二新卒って何歳まで？」よく聞かれるので答えます。
- 「履歴書に空白期間があるとアウトですか？」答えます。
- 「エージェントって怖くないですか？」正直に答えます。
