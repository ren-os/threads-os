# Threads投稿 生成プロンプト（50本バッチ）

## 使い方
このファイルをClaude Codeに渡すと、50本の投稿をCSV形式で生成できます。
生成後は `posts/threads_posts_stock.csv` に追記してください。

---

## プロンプト本文

以下の条件でThreads投稿を50本生成してください。

### アカウント設定
- アカウント名：ren_tenshoku_
- ターゲット：第二新卒・既卒・フリーター・転職初心者（18〜34歳）
- ゴール：転職タイプ診断LPへの流入 → LINE登録

### 投稿の導線
```
Threads投稿 → プロフィール → 転職タイプ診断LP → LINE登録
```

### 生成する投稿内訳（合計50本）
- 面接官シリーズ：10本
- 元人事シリーズ：10本
- ブラック企業シリーズ：10本
- 第二新卒シリーズ：10本
- フリーターシリーズ：10本

### 投稿設計ルール（必ず守ること）
1. 1行目のフック（hook）で読み手を止める
2. 直接売り込まない・案件名を出さない
3. URL直貼りは50本中5本まで（diagnosis_intent=highかつbodyにURL明記）
4. 基本CTAは「プロフィールへ」誘導
5. 不安を煽りすぎない・誇大表現をしない（「必ず」「絶対」「100%」禁止）
6. 1投稿は短め（Threads向け、3〜8行が理想）
7. 会話形式・ストーリー形式を多めにする
8. bodyの改行は `\n` で表現する（実際の改行文字は使わない）

### 使用するパターン
| パターン | 特徴 | フックの例 |
|---|---|---|
| 告白型 | 内部事情の告白 | 「正直に言います。」 |
| リスト型 | 箇条書き | 「〇〇のパターン。」 |
| 逆説型 | 常識を裏切る | 「〇〇は嘘です。」 |
| ストーリー型 | 実体験ナラティブ | 「〇〇した話をします。」 |
| 問いかけ型 | 読み手を主役に | 「〇〇してましたか？」 |
| 教え型 | 情報整理・解説 | 「〇〇の答えはここに集約されます。」 |
| Q&A型 | 検索意図に答える | 「〇〇、正直どう見ているか。」 |
| 共感型 | 感情に刺す | 「〇〇な気持ち、すごくわかります。」 |
| 許可型 | 罪悪感を解放 | 「〇〇していいです。」 |
| タイミング型 | 締め切り感 | 「今が〇〇のタイミングです。」 |
| 価値観型 | 思想の表明 | 「〇〇は古い。」 |
| 応援型 | 直接語りかけ | 「〇〇へ。」 |

### フィールド定義
| フィールド | 説明 |
|---|---|
| id | 連番（既存の最大IDの続きから） |
| category | 面接官 / 元人事 / ブラック企業 / 第二新卒 / フリーター |
| pattern | 告白型 / リスト型 / 逆説型 / ストーリー型 / 問いかけ型 / 教え型 / Q&A型 / 共感型 / 許可型 / タイミング型 / 価値観型 / 応援型 |
| hook | 1行目フック（ここで読み手を止める） |
| body | 本文（\nで改行。3〜8行推奨） |
| cta | 行動喚起テキスト |
| diagnosis_intent | high / medium / low |
| status | draft（固定） |
| created_at | YYYY-MM-DD（今日の日付） |

### diagnosis_intent の定義
- **high**：診断URLを明記 or 「プロフィールの診断」を強く示唆
- **medium**：「→ プロフィールへ」などの間接誘導
- **low**：CTAなし・認知拡大・共感メイン

### diagnosis_intent の配分目安（50本中）
- high：10本（各シリーズ2本）
- medium：30本（各シリーズ6本）
- low：10本（各シリーズ2本）

### CTAバリエーション
```
high系：
→ プロフィールの診断で整理できます
→ プロフィールに診断あります
→ プロフィールリンクから診断できます
（URL直貼り版）→ 転職タイプ診断はこちら👇\nhttps://ren-os.github.io/threads-os/career-diagnosis/

medium系：
→ プロフィールへ
→ まずはプロフィールを見てください
→ 詳しくはプロフィールへ
→ プロフィールに診断リンクあります

low系：
（空欄）
```

### 診断LP URL（URL直貼り投稿に使用）
```
https://ren-os.github.io/threads-os/career-diagnosis/
```

### CSV出力形式
```
id,category,pattern,hook,body,cta,diagnosis_intent,status,created_at
```

全フィールドをダブルクォートで囲んでください。
body内のダブルクォート文字（"）は "" でエスケープしてください。

## 追加生成時のチェックリスト

生成前に必ず確認：
- [ ] `posts/threads_posts_stock.csv` の最大IDを確認 → 続きの番号から採番
- [ ] 既存のhookと重複がないか確認
- [ ] URL直貼りが全体の10%以内に収まっているか
- [ ] diagnosis_intent の配分が high:medium:low = 2:6:2 になっているか（各シリーズ）
- [ ] 1シリーズ内でパターンが偏っていないか（同じパターンは連続させない）

## 投稿済み管理

投稿後は `posts/threads_posts_stock.csv` の status を `draft` → `posted` に変更。
または以下を実行：
```bash
python scripts/generate_threads_posts.py
```
（スクリプトでID指定してステータス更新可能）
