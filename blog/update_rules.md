# update_rules.md — リライト・更新ルール

記事の品質を維持・改善するためのリライト基準と情報更新ルールを定義する。

---

## リライトの基本方針

- リライトの目的は「順位を上げる」「CVRを改善する」の2つに絞る
- 感覚でリライトしない。kpi.md の数値を見て優先順位を決める
- 内容の「完全書き直し」と「部分修正」を区別して管理する

---

## リライト優先度の判断基準

### 最優先リライト（即対応）

| 条件 | 対応 |
|-----|------|
| 検索順位が10位以内 → 1ヶ月で圏外に落ちた | 競合との差分分析 → 見出し・文字数・構成を見直す |
| LINE登録率・CTRが突然低下した | CTAの位置・文章を見直す |
| エージェントの情報（料金・対応エリア等）が変更になった | 該当箇所を即時修正する |

### 定期リライト（月次・四半期）

| 条件 | 対応 |
|-----|------|
| 検索順位が11〜30位で停滞している | 見出し追加・内容充実・内部リンク強化 |
| 公開から3ヶ月以上経過して順位変動なし | タイトル・構成を競合上位と比較してリライト |
| フォーム・LINE登録数が増えた記事 | 成功パターンを他記事に横展開する |

---

## 順位変動時の対応フロー

```
[Googleサーチコンソールで順位確認（週次）]
    ↓
圏外（51位以下）→ コンテンツ品質を根本から見直す
11〜50位 → タイトル・見出し・内部リンクを強化する
4〜10位 → 競合上位と差分分析して上位コンテンツを超える
1〜3位 → CTAを最適化してCVRを上げることに集中する
```

## リライトの具体的な手順

### Step1：競合分析

1. 対象キーワードでGoogle検索を行う
2. 上位5記事の構成（タイトル・H2・文字数）を確認する
3. 自記事に不足しているH2・情報を洗い出す

### Step2：改善箇所の特定

| チェック項目 | 確認方法 |
|-----------|---------|
| タイトルに検索キーワードが含まれているか | タイトルを目視で確認 |
| 冒頭200字で検索意図に答えているか | 目視確認 |
| 文字数は上位記事と同等以上か | 上位記事の文字数と比較 |
| 内部リンクが3本以上あるか | 目視確認 |
| CTAが本文中2回以上・末尾1回あるか | 目視確認 |
| 画像にalt属性が設定されているか | HTMLソースで確認 |

### Step3：修正・更新

1. 修正箇所を書き出してから実施する（一気に変えすぎない）
2. 更新日を記事に反映する
3. Googleサーチコンソールで再インデックスをリクエストする

## 情報更新ルール

転職・エージェント情報は変化が速いため、以下のルールで最新化する。

| 更新対象 | 更新頻度 | 確認方法 |
|---------|---------|---------|
| 転職エージェントの料金・サービス内容 | 四半期（3ヶ月）ごと | 各エージェント公式サイトを確認する |
| エージェントの対応エリア・対応職種 | 四半期ごと | 同上 |
| 「2026年最新」など年度表記 | 年1回（1月） | タイトル・本文の年度を更新する |
| knowledge/asp.md の成果条件・報酬単価 | 変更があり次第 | ASP管理画面で確認 |
| knowledge/faq.md の情報 | 随時 | 相談事例・フォーム回答から更新 |

---

## リライト禁止事項

以下は順位が安定しているときは変更しない。

- 記事のURL（パーマリンク）→ 変更すると被リンク・評価がリセットされる
- 順位が上がっているときのタイトル変更 → 変更で順位が下がるリスクがある
- 大規模な削除（記事を大幅に短くする）→ 評価が下がる可能性が高い
