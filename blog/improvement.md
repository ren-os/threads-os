# improvement.md — ブログ改善サイクル

毎月のSEO分析・検索順位分析をもとに改善を実施し、
knowledge/ threads/ line/ forms/ にフィードバックするためのOS。

---

## 改善サイクルの全体像

```
[週次]
kpi.md でPV・順位・LINE登録率を確認
    ↓
アラートが出た記事を特定
    ↓
update_rules.md に沿ってリライト or 内部リンク強化

[月次]
月次サマリーをまとめる
    ↓
全体ファネルを確認（ブログ→LINE→相談→ASP）
    ↓
knowledge/ threads/ line/ forms/ にフィードバック
    ↓
翌月の記事テーマ・リライト優先度を決める
```

---

## 週次確認チェックリスト

毎週月曜日に以下を確認する。

```
【ブログ週次チェック】

□ Googleサーチコンソールで主要キーワードの順位変動を確認した
□ kpi.md の週次データを更新した（PV・UU・LINE登録数）
□ 順位が下落した記事を特定し、update_rules.md を確認した
□ 公開した記事から関連記事へのリンクを追加したか確認した
□ 新しく公開した記事はchecklist.md のチェックをすべて通過したか
```

---

## 月次SEO分析プロセス

### Step1：パフォーマンス確認

```
Googleサーチコンソールで確認：
・表示回数・クリック数・CTRの月次変化
・検索クエリ（意図しないキーワードで流入していないか）
・圏外に落ちた記事の特定

Googleアナリティクスで確認：
・月間PV・滞在時間・直帰率の変化
・LINE誘導クリック数の変化（short URLで計測）
・記事別のパフォーマンスランキング
```

### Step2：改善優先度の決定

update_rules.md のリライト優先度基準を使い、翌月のアクションを決める。

```
優先順位の考え方：
1. 圏外に落ちた記事（ASP直結記事は最優先）
2. 11〜30位で停滞している記事
3. 順位はいいがCTRが低い記事（タイトル改善）
4. 順位・PVはいいがLINE登録に結びついていない記事（CTA改善）
```

### Step3：翌月の新規記事テーマを決める

```
選び方：
・ideas.md の★★★（最優先）から選ぶ
・forms/responses.md のQ3（悩み）最多選択肢に対応したネタを選ぶ
・threads/posted_log.md で高評価だった投稿テーマをブログ記事に転用する
```

## knowledge/ へのフィードバック

| 更新先ファイル | フィードバック内容 | タイミング |
|------------|----------------|---------|
| knowledge/faq.md | 検索流入クエリから新しいよくある質問を追加する | 月次 |
| knowledge/objections.md | 記事コメント・FAQ検索クエリから不安・懸念を追加する | 月次 |
| knowledge/content_pillars.md | PVが高いカテゴリを優先テーマとして反映する | 四半期 |
| knowledge/asp.md | ASP成約実績をもとにエージェントの優先順位を更新する | 随時 |

### フィードバック例

```
「空白期間 転職 不安」というクエリで記事への流入が多い場合
→ knowledge/faq.md に「空白期間がある場合の転職の不安」のQ&Aを追加する
→ knowledge/objections.md に空白期間への反論設計を追加する
```
