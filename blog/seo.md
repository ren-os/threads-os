# seo.md — SEO設計ルール

記事のSEO品質を一定に保つためのルールブック。
Googleの検索意図・E-E-A-T・内部SEOの3軸で設計する。

---

## タイトル設計ルール

### 基本原則

- 文字数：30〜35文字（スマホ検索の表示上限に合わせる）
- 検索キーワードをタイトルの前半に入れる
- 「2026年最新」「完全ガイド」「体験談」など信頼・鮮度を示す言葉を入れる
- 数字を入れる（例：「5社比較」「3ステップ」「13の質問」）

### タイトルパターン

| パターン | テンプレート | 例 |
|---------|-----------|---|
| 比較 | 【2026年最新】〇〇 おすすめ〇選｜△△向け | 【2026年最新】第二新卒エージェント おすすめ5選｜フリーター・既卒も対応 |
| 体験談 | 〇〇が△△した体験談【〇〇で××した方法】 | フリーター26歳が正社員に転職した体験談【3ヶ月で内定が出た方法】 |
| 悩み解決 | 〇〇は△△？【正直に解説します】 | フリーター30歳はもう手遅れ？【正直に解説します】 |
| Q&A | 〇〇についてよくある質問〇個に答えます | 第二新卒についてよくある質問13個に答えます |
| ランキング | 〇〇におすすめの△△ TOP〇【選定基準あり】 | フリーターにおすすめの就職支援 TOP5【2026年最新】 |

### NG例

- 「転職について」（キーワードが曖昧）
- 「フリーターでも大丈夫！」（数字なし・感情訴求すぎる）
- 「第二新卒エージェントneoとは？転職エージェントのメリット・デメリットを解説！」（40字超え）

## 見出し（H2/H3）ルール

### H2の設計原則

- H2は4〜7個（多すぎると読まれない）
- H2に検索キーワードを1〜2個含める（自然な文章で）
- H2の順番は「結論 → 根拠 → 使い方 → FAQ」が基本
- 読者が「この見出しを読むだけで概要がわかる」ように書く

### H3の設計原則

- H2の補足・詳細に使う
- エージェントレビューでは「良い点・気になる点・向いている人」の3本が基本
- 1つのH2に H3 は2〜4個まで

### 見出し例（比較記事）

```
H2：結論：第二新卒にはこの3社がおすすめ
  H3：比較一覧表
H2：おすすめエージェントを選んだ基準
H2：1位：第二新卒エージェントneo の特徴・評判
  H3：良い点
  H3：気になる点
  H3：こんな人に向いている
H2：2位：就職カレッジ の特徴・評判
  （同上）
H2：エージェントの正しい使い方・選び方
H2：よくある質問（FAQ）
```

## 検索意図の分類と対応

Googleの検索意図は4種類。記事を書く前に必ず確認する。

| 意図タイプ | 説明 | 記事テンプレート |
|---------|------|-------------|
| Know（知りたい） | 情報収集・疑問解消 | Q&A記事（T05）・悩み解決（T04） |
| Do（やりたい） | 具体的な行動方法を探している | 体験談（T03）・転職進め方記事 |
| Go（行きたい） | 特定サービスを探している | エージェントレビュー・比較（T01/T02） |
| Buy（買いたい） | 申し込み・登録を検討している | 比較・ランキング（T01/T02） |

> GoとBuyは成約率が高いため優先的に記事を書く。

## 内部SEO（ページ最適化）ルール

| 項目 | ルール |
|-----|-------|
| メタタイトル | タイトルと同一（120文字以内） |
| メタディスクリプション | 120字以内・キーワード含む・読者のベネフィットを書く |
| 画像alt属性 | 画像の内容を説明するテキストを必ず設定（空白NG） |
| 内部リンク | 1記事あたり3本以上（internal_links.md参照） |
| URLスラッグ | 英数字・ハイフン区切り・キーワードを含む（例：freelance-to-seishain） |
| 文字数 | 3,000字以上（最低ライン）・上位記事±20%を目安 |
| 更新日 | 公開日と最終更新日の両方を記事に表示する |
| 表・リスト | H2に1つ以上は視認性の高い表・リスト・箇条書きを入れる |

## E-E-A-T対策（経験・専門性・権威性・信頼性）

Googleはコンテンツの信頼性を「E-E-A-T」で評価する。
転職・お金・仕事系（YMYL）は特に重要。

### Experience（経験）

- 実際の相談事例を記事に含める
- 「私が〇〇を使って感じたこと」「相談者Aさんのケース」を具体的に書く
- 数字・期間・具体的なエピソードを入れる（「3ヶ月で内定」「フリーター歴5年の方」）

### Expertise（専門性）

- 各記事は転職の特定テーマに集中させる（広く浅くNG）
- 専門用語を使う場合は必ず説明を添える
- knowledge/faq.md・knowledge/objections.md の知識を活用する

### Authoritativeness（権威性）

- プロフィールページにれんの経歴・活動実績を記載する（TODO：作成）
- Threadsアカウント（@kaishain_nigemichi）へのリンクを記事内に入れる
- 相談実績・内定事例を実績として記載していく

### Trustworthiness（信頼性）

- 情報の出典を明記する（厚労省データ・ASP公式情報等）
- 更新日を記事に表示する
- PRであることを適切に開示する（アフィリエイト記事には「PR」表記）
- 誇張表現・断言表現は避ける（「必ず」→「多くの場合」等）
