# affiliate_strategy.md — アフィリエイト戦略

ブログからASP経由でLINE登録・相談・成約を最大化するための設計。
knowledge/asp.md と連動して管理する。

---

## 収益の流れ

```
[ブログ記事（SEO流入）]
    ↓ 記事内のLINE誘導リンク or アフィリエイトリンク
    ↓
【ルートA】LINEに誘導 → 相談 → エージェント登録 → 内定 → ASP報酬
【ルートB】記事内のASPリンクを直接クリック → エージェント登録 → ASP報酬
```

> **優先ルートはA（LINE経由）**：個別対応することで成約率が高く、ブロック・離脱も防げる。
> ルートBは補助線。読者が「今すぐ自分で動きたい」場合のCTAとして設置する。

## CTA設計ルール

### CTAを入れる位置とタイプ

| 位置 | CTAタイプ | 目的 |
|-----|---------|------|
| 記事冒頭（最初のH2の前） | LINE誘導 軽め | 早期離脱ユーザーを捕捉する |
| 本文中（H2の2〜3個目後） | LINE誘導 or ASPリンク | 興味が高まったタイミングで行動を促す |
| 記事末尾 | LINE誘導（メイン）＋ASPリンク（サブ） | 記事を最後まで読んだ高意欲ユーザーを転換する |

### LINE vs ASP直接リンクの使い分け

| 状況 | 優先CTA |
|-----|--------|
| 悩み解決・体験談記事（Know/Do意図） | LINE誘導（相談につなぐ） |
| エージェントレビュー・比較記事（Go/Buy意図） | ASP直接リンク（今すぐ登録したい人向け）＋LINE誘導 |
| ランキング記事 | 各エージェントにASPリンク＋記事末尾にLINE誘導 |

## PR表記ルール（法的義務）

アフィリエイトリンクを含む記事には以下を必ず入れる。

```
【記事冒頭】
※ この記事にはプロモーション（広告）が含まれています。

【記事内のASPリンク近く】
（PR）
```

> 景品表示法・ステルスマーケティング規制に対応するため必須。
> 忘れた場合は即時追加すること。
