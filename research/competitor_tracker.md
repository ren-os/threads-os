# competitor_tracker.md — 競合バズ投稿・観測ログ

競合アカウントのバズ投稿・成長率・転用記録を蓄積する運用ファイル。
**アカウントのマスターリストは `research/competitor_list.md` を参照する。**

---

## このファイルの役割

```
competitor_list.md  → 「どのアカウントを見るか」のマスター管理
competitor_tracker.md → 「見て何が起きたか」の観測ログ・分析記録

毎朝のリサーチ手順：
1. competitor_list.md を開いてアカウント一覧を確認する
2. 各アカウントの最新投稿を確認する
3. バズ投稿・気づきをこのファイルに記録する
```

---

## 転用ルール（再掲）

```
1. フックをそのままコピーしない（言い換える）
2. 型・構成パターンは参考にしてOK
3. テーマは同じでも「れんの体験談・言葉」で書く
4. 転用した場合は上記「転用ログ」に必ず記録する
5. 同じネタを2週間以内に2回使わない
```

## 週次確認チェックリスト

```
□ competitor_list.md を開いてアカウント一覧を確認したか
□ TH-01・TH-02（毎日確認対象）を今週毎日確認できたか
□ 新たなバズ投稿はあったか → 伸びた投稿ログに記録したか
□ viral_archive.md への詳細記録は済んでいるか
□ 転用できるアイデアはあったか → threads/ideas.md に追加したか
□ 競合が新しいCTA方法・導線を使い始めていないか確認したか
□ competitor_list.md の「最終確認日」を更新したか
□ 月次：competitor_list.md のフォロワー数欄を更新したか
```
