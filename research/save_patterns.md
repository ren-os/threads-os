# save_patterns.md — 保存される投稿の特徴・構造分析

「保存」はThreadsアルゴリズムで長期リーチに最も効くシグナル。
保存されやすい投稿の共通構造を分析し、投稿生成に反映する。

---

## なぜ保存されるのか

```
保存 = 「後で見返したい」という行動
= 情報として価値があると判断された

保存されやすい投稿の共通点：
① 具体的な手順・チェックリストが書いてある（後で使いたい）
② 数字・固有名詞があって信頼性が高い（保存して誰かに見せたい）
③ 「自分の状況に当てはまる」コンテンツ（後で読み返したい）
④ 見返したくなるほど情報が詰まっている（長文・まとめ系）
```

## プラットフォーム別・保存パターン分析

### Threads

```
保存されやすい型：
1位：T02 チェックリスト型（「□ 〇〇を確認する」形式）
2位：T07 知らないと損するシリーズ（実用情報）
3位：T03 NG vs OK 比較型（すぐ使える対比）
4位：T04 ランキング型（情報の整理）
5位：T09 ビフォーアフター型（Before/After の対比）

保存されにくい型：
- T06 共感一言型（感情的価値はあるが情報としての価値は低い）
- T10 CTA型単体（情報なしのCTAは保存されない）

保存されやすいフックの特徴：
□ チェックリスト予告「確認してほしい○つ」
□ 「知らないと損する」系
□ 「まとめます」「整理します」系
□ 数字が入っている「〇〇選」「〇〇の〇つ」
```

## 保存される投稿の共通構造テンプレート

これらのテンプレートは generate_posts.md の生成時に参照する。

### 保存テンプレート A：チェックリスト型

```
[フック：「〇〇する前に確認してほしいことがあります。」]

□ [項目1]（なぜ重要か1行で）
□ [項目2]（なぜ重要か1行で）
□ [項目3]（なぜ重要か1行で）
□ [項目4]（なぜ重要か1行で）

[まとめ：「これを知っていると〇〇が変わります。」]

→ 保存されやすい理由：後で実際に使うために保存される
```

### 保存テンプレート B：知らないと損するシリーズ

```
知らないと損する転職の話。

[テーマを一言で]

NG：「〇〇してしまうパターン」
↓
OK：「〇〇する方法」（具体的に）

[まとめ：「これを知っているかどうかで、結果が変わります。」]

→ 保存されやすい理由：実用情報として後で使うために保存される
```

### 保存テンプレート C：まとめランキング型

```
[ジャンル]のよくある[質問/ミス/特徴]、まとめます。

①〇〇
→ [理由・解説を1行で]

②〇〇
→ [理由・解説を1行で]

③〇〇
→ [理由・解説を1行で]

特に②が重要で、[補足1〜2行]

→ 保存されやすい理由：情報がランク付けされて整理されているため
```

### 保存テンプレート D：ビフォーアフター型

```
[フック：変化の予告]

【Before】
[問題のある状態を具体的に]

↓ やったこと（1〜3つ）

【After】
[改善した結果を具体的に・数字あれば]

[学び：「変えたのは〇〇だけでした。」]

→ 保存されやすい理由：「自分もやってみよう」と思い保存される
```

## 保存率改善ルール

```
週次チェック（月曜のルーティン内）：
□ 前週の投稿で「保存数が多かった投稿」はあったか
□ 保存された投稿の型は何か → 週次で上位型を特定する
□ 保存されなかった投稿との違いは何か

改善アクション：
・保存率が低い週 → 翌週はチェックリスト型・知らないと損するシリーズを増やす
・保存率が高い型が特定できたら → threads/viral_patterns.md に追記する
・保存率の低い型 → 生成比率を下げる（generate_posts.md の条件シートで調整）
```
