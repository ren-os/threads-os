# hooks_archive.md — 伸びたフックアーカイブ

SNS4プラットフォームで「スクロールを止めた冒頭1行」を蓄積するDB。
投稿生成時（generate_posts.md Step1）に必ず参照し、高パフォーマンスフックの構造を反映する。

---

## フックとは

```
投稿の冒頭1〜2行。これだけで「読むか/スキップするか」が決まる。
→ 伸びたフックの「構造」を分析して、れんの投稿に転用する
→ コピーは禁止。「型・感情・言葉の選び方」を参考にする
```

## 感情タイプ別フック分類

### 共感型（「自分のことだ」と思わせる）

```
パターン例：
「〇〇、怖いですよね。」
「〇〇、わかりますか。」
「〇〇と思ってる人へ。」

れんの転用テンプレート：
「[ターゲットの心の声]、[共感の言葉]。」
```

### 数字型（具体性で信頼させる）

```
パターン例：
「[数字]ヶ月で内定が出た話をします。」
「[数字]社落ちた後に変えたこと。」
「[数字]つのことだけで変わった話。」

れんの転用テンプレート：
「[数字][時間/回数][結果]、[話題の予告]」
```

### 逆説型（「え？」と思わせる）

```
パターン例：
「転職は諦めた方がいい人の特徴、正直に書きます。」
「エージェントを使わない方がいいケース、あります。」
「短期離職が武器になる場合があります。」

れんの転用テンプレート：
「[意外な主張]、[正直に/あります/書きます]。」
```

### 問いかけ型（「自分はどうだろう」と考えさせる）

```
パターン例：
「これ、1つでも当てはまる人いませんか？」
「〇〇な人、この後が大事です。」
「〇〇をやっていない人、もったいないです。」

れんの転用テンプレート：
「[条件]な人、[行動喚起/メッセージ]。」
```

### 情報開示型（「知りたい」と思わせる）

```
パターン例：
「知らないと損する転職の話。」
「〇〇でよく聞かれること、ぶっちゃけます。」
「面接官が実際に見ていること、教えます。」

れんの転用テンプレート：
「知らないと損する[テーマ]の話。」
```

## knowledge/hooks.md との連携ルール

```
このアーカイブで「月に2回以上バズったフック構造」が見つかった場合：
→ knowledge/hooks.md の「未使用フックストック」に転用案を追加する

れんの投稿で使い終わったフックは：
→ knowledge/hooks.md の「使用済みフック」に移動する
→ このアーカイブの「れん転用済み」欄を更新する

フック構造の更新タイミング：
→ 週次（月曜のweekly.mdルーティン内で実施）
```

## れんが使えるフック転用ストック

research で発見したフックを「れんが使える形」に変換して蓄積するセクション。

