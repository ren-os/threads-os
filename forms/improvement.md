# improvement.md — フォーム改善サイクル

フォームの回答データをもとに毎週改善を回し、
knowledge/ threads/ line/ にフィードバックするためのOS。

---

## 改善サイクルの全体像

```
[週次]
kpi.md の数値を確認
    ↓
conversion.md でファネルの詰まりを特定
    ↓
改善アクションを1〜2つ実施
    ↓
翌週のkpi.md で効果を検証

[月次]
responses.md の回答傾向を分析
    ↓
knowledge/ threads/ line/ にフィードバックする
    ↓
improvement.md に記録する
```

---

## 週次確認チェックリスト

毎週月曜日に以下を確認する。

```
【フォーム週次チェック】

□ kpi.md の週次データを入力した（F1/F3/F6）
□ 回答完了率を確認した（アラート：35%未満）
□ 相談予約率を確認した（アラート：30%未満）
□ フォーム送信後の返信漏れがないか確認した
□ line/kpi.md と数値を照合した
□ threads/kpi.md のThreads指標と連動を確認した
```

---

## 月次改善プロセス

### Step1：回答データの分析（responses.md を使う）

```
確認すること：
① Q1（雇用形態）の分布 → ターゲット属性に合っているか？
② Q3（悩み）の分布 → 投稿ネタと合っているか？
③ Q4（希望時期）の分布 → 温度感が想定と合っているか？
④ Q5（自由記述）のキーワード → 新しいインサイトがあるか？
```

### Step2：改善アクションの選定

conversion.md の施策一覧から実施するものを選ぶ。

優先順位：
1. アラートが出ている指標を改善する（kpi.md のアラートルール参照）
2. 全体ファネルの最も弱いポイントを改善する
3. 余裕があればA/Bテストを追加する

### Step3：各ファイルへのフィードバック

回答データをもとに以下のファイルを更新する。

---

## knowledge/ へのフィードバック

| 更新先ファイル | フィードバック内容 | タイミング |
|------------|----------------|---------|
| knowledge/target.md | Q1・Q2の分布でターゲットペルソナの精度を上げる | 回答50件ごと |
| knowledge/content_pillars.md | Q3（悩み）の最多選択肢に合わせてコンテンツ優先度を調整する | 月次 |
| knowledge/faq.md | Q5の自由記述から新しいよくある質問を追加する | 月次 |
| knowledge/objections.md | Q3・Q5から新しい不安・懸念を抽出し追加する | 月次 |

### フィードバック例

```
Q3「自分の経歴・スキルで転職できるか不安」が全体の40%を占めた場合
→ knowledge/content_pillars.md の優先テーマに
  「経歴への不安を解消するコンテンツ」を追加する
→ knowledge/objections.md に「経歴が不安な人への反論設計」を追加する
```

## threads/ へのフィードバック

| 更新先ファイル | フィードバック内容 | タイミング |
|------------|----------------|---------|
| threads/ideas.md | Q3・Q5の実際の悩みをネタとして追加する | 月次 |
| threads/hooks.md | フォームでよく出るキーワードをフックに転用する | 月次 |
| threads/posted_log.md | フォーム流入が増えた時期の投稿を高評価として記録する | 随時 |

```

→ threads/ideas.md の「優先度：高」に
  「空白期間1年でも正社員になった話」のネタを追加する
→ threads/hooks.md に
  「空白期間1年の私が〇〇した話」のフック案を追加する
```

## line/ へのフィードバック

| 更新先ファイル | フィードバック内容 | タイミング |

