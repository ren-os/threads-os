# conversion.md — フォーム転換率改善システム

フォーム表示→回答開始→回答完了→相談予約 の各転換率を管理・改善するOS。
line/conversion.md と連動してファネル全体を最適化する。

---

## 管理する転換ポイント

```
[LINE内CTAタップ（フォームURL）]
    ↓ [転換率A] フォーム回答開始率
[Q1回答]
    ↓ [転換率B] フォーム回答完了率（Q1→送信完了）
[送信完了]
    ↓ [転換率C] 相談予約率（送信完了→日程確定）
[相談予約確定]
```

---

## 転換率の目標値

| 転換ポイント | 現状 | フェーズ1目標 | フェーズ2目標 |
|-----------|------|------------|------------|
| A. 回答開始率 | TODO | 60%以上 | 75%以上 |
| B. 回答完了率 | TODO | 55%以上 | 65%以上 |

> 解釈：100人がフォームURLをタップ→17人が相談予約まで進む（フェーズ1目標）

## 転換率A：回答開始率の改善

### 低下する主な原因

| 原因 | 対策 |
|------|------|
| CTAコピーが弱い | 「30秒で入力できます」「無料・強引な勧誘なし」を必ず入れる |
| フォームのリンクプレビューが地味 | Googleフォームのヘッダー画像を設定する（Canva等で作成） |
| フォームが重くて開かない | 画像を軽量化する |
| LINE内で流れて見逃す | step_messages.md のDay5・Day7でフォームリンクを再掲する |

### CTAコピーの改善テンプレート

```
【現行A案】
以下のフォームに答えてもらえれば一緒に考えます😊
👇 30秒で入力できます
[フォームURL]

【改善B案（フック強化）】
転職に迷っているなら、まずここから。
👇 30秒・5問・完全無料
[フォームURL]
「何から話せばいいかわからない」という方でもOKです。

【改善C案（安心訴求）】
勧誘なし・費用ゼロ・話すだけでもOK。
まずあなたの状況を教えてください。
👇
[フォームURL]
```

> A/Bテストの実施はforms/kpi.md で管理する。

## 転換率B：回答完了率の改善

### 低下する主な原因と対策

| 原因 | 対策 |
|------|------|
| 質問数が多い | 5問を上限とし、それ以上は追加しない（logic.md参照） |
| 自由記述でつまる | Q5を「任意」にする・ガイドテキストを入れる（questions.md参照） |
| 「送ったら何が起きる？」不安 | フォーム冒頭と各質問の間に安心文を入れる |
| スマホで見にくい | Googleフォームのモバイルプレビューで必ず確認 |

### フォーム内の安心文（各セクション挿入用）

```
フォーム冒頭：
「このフォームの回答はれんだけが確認します。
 第三者に共有することはありません。」

Q3の上：
「転職に迷っている段階でも大丈夫です。
 一番近い選択肢を選んでください。」

Q5の上：
「（任意）書けそうなことがあれば教えてください。
 空白のまま送信してもOKです。」
```

## 転換率C：相談予約率の改善

### 低下する主な原因と対策

| 原因 | 対策 |
|------|------|
| フォーム送信後のLINE返信が遅い | 送信後24時間以内に必ず返信する（line/conversion.md参照） |
| 返信がテンプレすぎる | Q5の自由記述に触れた一言を必ず入れる（個人感を出す） |
| 相談形式が怖い | 「話すだけでもOK」「断ってもOK」を返信に入れる |
| 日程候補の調整が面倒 | 候補日を5つ以上出す／Calendlyなどの日程調整ツールを検討する |

### 送信後返信テンプレート（最適化版）

```
[名前]さん、フォームの回答ありがとうございます！

[Q5 or Q3の内容への個別一言コメント]

相談の日程を決めさせてください。
以下のどれかご都合はどうですか？

・[日時候補1]
・[日時候補2]
・[日時候補3]
・[日時候補4]
・[日時候補5]

合わなければ他の日程も調整できます😊
```
